Twig

Gameforest includes several html templates and source code that compiled by twig. Twig.js is a pure JavaScript implementation of the Twig PHP templating language.

How to use

We can access twig files in ./src/twig directory. After we have made new modifications in source then need to run a task to compile twig files into html.

yarn html

It will compile our twig files into the ./dist directory as pure and clean html files.

Extends

With extends we can add base html attributes to our twig file like header or footer so required to include it in every single twig file at top of the page.

{% extends "base.twig" %}

CSS block

Add your custom css or 3rd party plugin stylesheets by using block css just like in the example below and it will be displayed in the html header.

{% block css %}
    <link rel="stylesheet" href="...">
{% endblock %}

Javascript block

Add your custom javascript or 3rd party plugin javascripts by using block js just like in the example below and it will be displayed at the bottom of page.

{% block js %}
    <script src="..."></script>
{% endblock %}

Body block

Use body block to display sections and components inside html body attribute just like in the example below.

{% block body %}
    <section>
        <div class="container">
            <div class="row">
                <div class="col">
                    ...
                </div>
            </div>
        </div>
    </section>
{% endblock %}

Twig markup

Basic example to get start to build your own twig page.

{% extends "base.twig" %}

{% block body %}
    <section>
        <div class="container">
            <div class="row">
                <div class="col">
                    ...
                </div>
            </div>
        </div>
    </section>
{% endblock %}

JSON data

Json data is a great way if we would like to define a global variable like sitename or just add posts etc...

{
    "config": {
        "title": "Gameforest Bootstrap Gaming Theme",
        "brand": "Gameforest",
        "version": "1.0",
        "navbar": "fixed",
        "footer": "static",
        "dateformat": "F d, Y"
    }
}

Variable

A variable can be set on every page like in the example below. In this case we would like to set a sticky footer for our page so we define page variable and the footer key.

{% set docs = {
    footer: 'sticky'
} %}
option value default description
title string empty Meta title for the current page.
description string empty Meta description for the current page.
navbar string fixed Set requested header for the page by using: static, fixed value.
footer string static Set requested footer for the page by using: sticky, static value.
