Sass

Sass is an extension of CSS, adding nested rules, variables, mixins, selector inheritance, and more. It's translated to well-formatted, standard CSS using the command line tool or a web-framework plugin.

Introduction

If you have never used SCSS before, check out the language features and documentation. Now, theming is accomplished by Sass variables, Sass maps, and custom CSS. When working with the Gameforest SCSS sources, We have a few recommendations.

Customization

Whenever possible, avoid modifying Gameforest’s core files. For Sass, that means creating your own stylesheet that imports Gameforest so you can modify and extend it. Scss files are included in src folder like this:

gameforest/
└── src/
    └── scss
        └── custom
            └── _theme.scss
            └── _variables.scss

Add your custom variables to the custom/_variables.scss file and custom classes or imports from custom/_theme.scss file. When a new update has been released then don't overwrite these files.

Variables

A lot of customization is possible by simply overwriting the values of already declared variables. You can find all variables for each component inside their SCSS files of the variables.scss file and override them in your theme. You can also add your own custom variables by adding it to custom/_variables.scss

gameforest/
└── src/
    └── scss
        └── custom
            └── _variables.scss
        └── theme
            └── _variables.scss

We can find pre defined variables in the _variables.scss just like below.

// Primary Color
$primary: #2575dc !default;

If we would like to modify a pre defined variable then just look for the right variable in _variables.scss and add it with choosen option to the _custom.scss file without the !default parameter like this:

// Primary Color
$primary: #6db124;

After save we are ready to run the gulp task and css files will be modified.


Notice: in order to see the result of modification what you have made before, it's required to run gulp task. Learn more about gulp go the dedicated page that can be found in the sidebar.

Plugins

It's fine when you would like to use custom 3rd party plugins with scss and node_modules. Below you can find an example how to import plugins scss as a node_module.

@import "../../node_modules/owl.carousel/src/scss/owl.carousel";

It is also required to add the plugin to the dependencies by using yarn or npm install command in terminal.
